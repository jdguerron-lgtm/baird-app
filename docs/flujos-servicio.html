Flujo de Garantia

Servicios pagados por la marca (ej. Mabe/GE). El cliente no paga nada. El tecnico cobra tarifa fija segun complejidad.

1

Solicitud de servicio

Cliente Admin (carga masiva) Sistema
El cliente llena el formulario en /solicitar o el admin sube un Excel (.xlsx formato BITACORA) en /admin/carga-masiva. El sistema guarda en Supabase, envia confirmacion por WhatsApp al cliente, y notifica a tecnicos compatibles.
API
POST /api/solicitar o POST /api/carga-masiva
Estado
pendiente → notificada
WhatsApp al cliente
Confirmacion de solicitud
WhatsApp al tecnico
nueva_solicitud_v3

2

Aceptacion del tecnico

Tecnico Sistema
El tecnico hace clic en "Aceptar" en WhatsApp. La pagina /aceptar/{token} hace un UPDATE atomico (el primero en aceptar gana). Si otro tecnico intenta despues, recibe "servicio ya tomado".
Pagina
/aceptar/{token}
Estado
notificada → asignada
WhatsApp al tecnico
servicio_asignado_tecnico_v3
WhatsApp al cliente
tecnico_asignado_cliente_v4
↓
3

Diagnostico + Reparacion inmediata

Tecnico Sistema
El tecnico visita al cliente, diagnostica el problema, y llena el formulario de tarifa de garantia (codigo de complejidad, bono incentivo, codigo de falla). Luego procede directamente a reparar.
Pagina
/tecnico/{token}/diagnostico/{id}
API
POST /api/diagnostico (branch garantia)
Estado
asignada → en_proceso
WhatsApp al cliente
Texto: "El tecnico esta trabajando en la reparacion"
↓

El tecnico sube fotos del equipo reparado, completa el checklist de calidad, firma digital, y confirma la ubicacion GPS.

Confirmacion del cliente

Cliente
El cliente recibe un WhatsApp con boton para confirmar. Al hacer clic, ve la pagina de confirmacion donde puede marcar "Satisfecho" o "Reportar problema".

Flujo Particular (No-Garantia)

Servicios pagados por el cliente. Tarifa de diagnostico fija ($80,000 COP) con anticipo del 50% ($40,000 COP). Despues del diagnostico, el tecnico envia cotizacion que el cliente debe aprobar.

1

Solicitud de servicio

Cliente Sistema
El cliente llena el formulario en /solicitar con es_garantia = false. El sistema guarda en Supabase, envia WhatsApp con tarifa de diagnostico y anticipo, y notifica a tecnicos compatibles.
API
POST /api/solicitar
Estado
pendiente → notificada
WhatsApp al cliente
solicitud_particular_cliente_v1
Incluye: tarifa diagnostico $80,000 + anticipo $40,000
WhatsApp al tecnico
solicitud_particular_tecnico_v1
Incluye: pago diagnostico para el tecnico
↓
2

Aceptacion del tecnico

Tecnico Sistema
Igual que garantia: clic en "Aceptar", UPDATE atomico, primer tecnico gana. Pero el estado siguiente es diferente: diagnostico_pendiente en vez de asignada.
Pagina
/aceptar/{token}
Estado
notificada → diagnostico_pendiente
WhatsApp al tecnico
servicio_asignado_tecnico_v3
WhatsApp al cliente
tecnico_asignado_particular_v1
Incluye: info tecnico + tarifa + anticipo
↓
3

Diagnostico + Cotizacion

Tecnico Sistema
El tecnico visita al cliente, diagnostica el problema, y genera una cotizacion con mano de obra y repuestos. El sistema envia la cotizacion al cliente por WhatsApp con un boton para aprobar.
Pagina tecnico
/tecnico/{token}/diagnostico/{id}
(muestra campos de cotizacion, no tarifa)
API
POST /api/diagnostico (branch particular)
Estado
diagnostico_pendiente → cotizacion_enviada
WhatsApp al cliente
cotizacion_cliente_v1
Incluye: diagnostico, mano obra, repuestos, total + boton "Aprobar"
↓
4

Aprobacion de cotizacion (paso extra)

Cliente Sistema
El cliente abre /cotizacion/{token} donde ve el diagnostico, fotos de evidencia, desglose de costos, y dos botones: Aprobar o Rechazar. Si aprueba, el tecnico recibe notificacion para proceder. Si rechaza, el servicio se cierra.
Pagina cliente
/cotizacion/{token}
API
POST /api/aprobar-cotizacion
Si APRUEBA
cotizacion_enviada → cotizacion_aprobada → en_proceso
cotizacion_aprobada_tecnico_v1 al tecnico
Si RECHAZA
cotizacion_enviada → cotizacion_rechazada
Texto de rechazo al tecnico
↓
5

Reparacion + Completacion

Tecnico
Identico al flujo de garantia desde este punto: el tecnico repara, sube fotos, completa checklist, firma digital.
Estado
en_proceso → en_verificacion
WhatsApp
confirmar_servicio_v3 al cliente
↓
6

Confirmacion del cliente

Cliente
Identico al flujo de garantia: el cliente confirma satisfaccion o reporta problema.
Estado final
completada o en_disputa
Pago tecnico
Total cotizado (mano obra + repuestos)

Datos JSONB en solicitudes_servicio

triaje_resultado (Garantia)

  • diagnostico_tecnico: texto
  • complejidad: nivel
  • codigo_complejidad: codigo TSS
  • tarifa_mano_obra: monto
  • bono_incentivo: monto
  • total_servicio: monto
  • codigo_falla: objeto falla

cotizacion (Particular)

  • diagnostico_tecnico: texto
  • mano_obra: monto
  • repuestos: monto
  • total: mano_obra + repuestos
  • token: UUID aprobacion
  • cotizado_at: timestamp
  • aprobado_at / rechazado_at

Modelo de pago

Garantia

  • La marca paga a Baird Service
  • Tarifa fija segun codigo complejidad
  • Bono incentivo incluido
  • Cliente NO paga nada
  • pago_tecnico = totalServicio

Particular

  • Cliente paga a Baird Service
  • Diagnostico: $80,000 COP (fijo)
  • Anticipo: 50% = $40,000 COP
  • Reparacion: segun cotizacion
  • pago_tecnico = cotizacion.total

Reglas importantes de WhatsApp

Ventana de 24 horas: Los mensajes de texto libre solo se pueden enviar si el cliente envio un mensaje al negocio en las ultimas 24 horas. Los templates se pueden enviar en cualquier momento.

Nombres de templates: Deben coincidir exactamente con lo aprobado en Meta Business Manager. Si se elimina un template, hay un cooldown de 4 semanas antes de reusar el mismo nombre.
